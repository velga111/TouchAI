TouchAI 通过 `bash` 工具，帮助你高效处理本地任务与终端操作。

## 它能帮你做什么

- 读取项目中的代码、配置和日志文件
- 执行终端命令并返回输出结果
- 根据报错、测试结果或日志继续分析问题
- 在已有结果基础上继续给出下一步建议

## 什么时候用它

- 运行测试并解释失败原因
- 查看当前分支改动、提交记录或构建日志
- 安装依赖、启动项目、执行脚本
- 结合错误输出定位更可能的问题点

## 你可以怎么提需求

直接按任务来描述即可，例如：

- 帮我运行这个项目的测试，并解释失败原因
- 帮我查看当前分支改了什么
- 帮我安装依赖并启动开发环境
- 帮我根据报错信息定位问题

## 审批与边界

终端命令的执行是否需要你逐次确认，取决于当前的内置工具设置。使用时建议注意：

- 执行前先确认命令用途是否清楚
- 删除、覆盖、重置等高风险命令要格外谨慎
- 如果项目环境、本地权限或工作目录不完整，命令可能无法成功执行

这项能力的重点是“协助你推进任务”，不是无边界地执行任何命令。

## 一个典型流程

1. 先告诉 TouchAI 你想完成什么，例如“运行测试并解释失败原因”
2. TouchAI 读取相关文件
3. TouchAI 执行所需命令并返回输出结果
4. 它根据结果解释问题，并给出下一步建议
5. 你再决定是否继续修复、验证或切换模型

## 常见问题

### 为什么命令没有直接执行？

有些环境会要求在执行前先审批，这是为了避免误操作或高风险命令直接落地。

### 为什么命令执行失败？

常见原因包括：

- 当前工作目录不对
- 缺少依赖或运行环境
- 本地权限不足
- 命令本身就有错误

### 我应该先让它读文件，还是直接让它跑命令？

如果你已经知道要执行什么，可以直接描述任务。  
如果你还不确定问题位置，通常先让它读代码或日志，再运行命令会更稳。
