在 TouchAI 中添加、连接并验证一个 MCP 服务的完整流程。

## 1. 寻找服务

先在 MCP 商店里查找可用服务；如果你已经知道要接入哪个服务，也可以直接查看它的官网或官方文档，确认接入方式和所需参数。
- [mcp.so](https://mcp.so/)
- [Smithery](https://smithery.ai/)
- [Glama MCP](https://glama.ai/mcp/servers)
- [MCP 社区](https://mcp.community/)

## 2. 打开 MCP 工具页

进入应用 **设置**，在左侧导航中打开 **MCP 工具**。

![打开 MCP 工具页](/extensions/mcp-add-service/mcp-open-tool-page.png)

## 3. 新建服务器

点击左下角的 **+ 添加 MCP 服务器**。

![新建 MCP 服务](/extensions/mcp-add-service/mcp-new-service.png)

## 4. 填写配置

在 **配置** 页签中，按目标服务文档填写参数。

通常你至少会用到这些内容：

- **名称**
- **传输类型**
- **命令** 或 **URL**
- 如果服务要求认证，再补充 **请求头** 或 **环境变量**
- 如果服务有特殊启动要求，再补充 **参数** 或 **工作目录**

![添加 MCP 服务表单](/extensions/mcp-add-service/mcp-add-service-form.png)

## 5. 查看工具

完成配置后，切到 **工具** 页签，查看这个服务有什么工具可用。

![查看 MCP 服务工具](/extensions/mcp-add-service/mcp-view-service-tools.png)

## 6. 在对话里使用服务

连接成功后，你就可以在 TouchAI 中使用这个服务提供的能力，把真实任务接进当前对话。

![使用 MCP 服务](/extensions/mcp-add-service/mcp-use-service.png)

## 7. 查看日志

如果连接或调用过程中有异常，可以到 **日志** 页签查看服务日志，帮助你定位问题。

![查看 MCP 服务日志](/extensions/mcp-add-service/mcp-view-service-logs.png)

## 常见问题

### 为什么服务连接不上？

- 检查地址或启动命令是否填写正确
- 查看所需密钥、令牌或环境变量是否完整
- 确认当前网络或本地依赖是否可用
- 查看官方文档确认服务本身是否支持当前接入方式

### 为什么工具列表是空的？

可能是服务尚未成功连接，或者配置虽然保存了，但关键参数还不完整。

### 日志里有报错时该怎么看？

先看是“连接失败”“认证失败”还是“工具执行失败”，再回到配置页逐项检查对应参数。
