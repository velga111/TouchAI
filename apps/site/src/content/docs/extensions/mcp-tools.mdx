MCP 扩展可以把外部服务接入 TouchAI，让它不只处理本机文件和通用任务，还能继续访问代码托管平台、知识库、数据库、浏览器自动化和更多业务系统。

## 什么时候该用 MCP

当你的任务发生在这些外部平台里时，通常就该考虑使用 MCP：

- 代码托管平台，如 GitHub、GitLab
- 文档与知识库，如企业 Wiki、在线文档、笔记系统
- 浏览器和网页自动化场景
- 数据库、报表和内部业务系统
- 团队协作与办公服务

如果任务主要发生在当前电脑、本地文件或本地项目里，通常先用 [内置工具](/features/built-in-tools/) 就够了。

## MCP 和内置工具有什么区别

| 场景 | 更适合内置工具 | 更适合 MCP |
| --- | --- | --- |
| 本机文件、终端、本地任务 | 是 | 否 |
| 外部平台、云文档、数据库、业务系统 | 否 | 是 |
| 开箱可用 | 是 | 否，需要先添加服务 |
| 任务范围 | 通用桌面任务 | 特定平台或业务流程 |

## 从哪里开始

- 如果你已经知道要接什么服务：直接看 [添加服务](/extensions/mcp-add-service/)
- 如果你还不确定应该接哪一类服务：先看 [热门服务](/extensions/mcp-popular-scenarios/)
- 如果你其实只是想找本地文件或执行终端命令：回到 [内置工具](/features/built-in-tools/)
