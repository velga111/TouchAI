## What this capability is

When the current model is no longer a good fit for the task at hand, you can use model upgrade to continue the same task on a stronger model in the upgrade chain instead of starting a brand-new conversation.

This is especially useful when:

- A simple question gradually turns into a more complex analysis
- The current context becomes much larger and the original model starts to struggle
- You want to continue the same task instead of switching models manually and starting over

## How to use it / A typical flow

### Configure the upgrade chain

First configure the upgrade chain in the built-in tools area of settings.

![Upgrade chain](/features/upgrade-model/upgrade-model.png)

### Use it

Model upgrade switches according to the configured upgrade chain:

1. If the current model is already in the chain, it upgrades to the next stronger model after it
2. If the current model is not in the chain, it tries to move back to the first suitable available model in the chain
3. If the current model is already at the end of the chain, or there is no stronger model in the chain, the upgrade fails

![Upgrade success](/features/upgrade-model/upgrade-model-success.png)

A typical flow usually looks like this:

1. Configure the upgrade chain ahead of time
2. Trigger model upgrade when the current model starts to feel insufficient
3. Let TouchAI continue the same task on the next suitable model

## Common questions

### Why can the model not be upgraded?

The most common reasons are:

- The upgrade chain has not been configured yet
- There is no available model in the upgrade chain
- The current model is already the last step in the chain

### Will upgrading interrupt the current task?

The point of this capability is to keep working in the current task context instead of forcing you to describe all of the background again.

### Does upgrading permanently replace the model I am using?

Upgrading changes the model used for the rest of the current task.  
If you want to stay on a specific model, you should still confirm it manually in model settings or in the current session.
